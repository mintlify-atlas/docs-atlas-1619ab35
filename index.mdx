  <div className="text-center mb-12">
    <h2 className="text-3xl font-semibold text-gray-900 dark:text-white mb-4">Quick start</h2>
    <p className="text-base text-gray-600 dark:text-gray-400 max-w-2xl mx-auto">
      Get TCP Streamer running in minutes. Follow these steps to start streaming audio to your TCP server.
    </p>
  </div>

  <Steps>
    <Step title="Download and install">
      Download the latest release for your platform from the [releases page](https://github.com/NaturalDevCR/TCP-Streamer/releases).
      
      - **macOS**: Open the `.dmg` file and drag to Applications
      - **Windows**: Run the `.msi` or `.exe` installer
      - **Linux**: Install the `.AppImage` or `.deb` package
    </Step>
    
    <Step title="Select your audio source">
      Launch TCP Streamer and choose the audio input device you want to stream.
      
      For system audio capture:
      - **Windows**: Enable **"Loopback Mode (Windows)"** to capture system audio via WASAPI
      - **macOS**: Use [BlackHole](https://github.com/ExistentialAudio/BlackHole) or similar virtual audio device
      - **Linux**: Configure PulseAudio or ALSA loopback
    </Step>
    
    <Step title="Configure your destination">
      Enter your TCP server details:
      
      ```bash Example Configuration
      IP Address: 192.168.1.100
      Port: 4953
      Sample Rate: 48000 Hz
      Buffer Size: 1024 samples
      ```
      
      <Note>
        For Snapcast servers, use port **4953** (default TCP streaming port).
      </Note>
    </Step>
    
    <Step title="Start streaming">
      Click **Start Streaming** and monitor the real-time statistics:
      
      - Connection status and uptime
      - Current bitrate and data sent
      - Network quality score and jitter
      - Buffer health and latency
      
      Enable **Auto-reconnect** to maintain connection stability, and **Auto-start on launch** to run at system boot.
    </Step>
  </Steps>
</div>
